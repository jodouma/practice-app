# Guide étudiant — Application d'entraînement

## Pourquoi utiliser cette application ?

Cette application vous aide à préparer l'examen de `Contrôle d'accès et gestion d'identité` avec :
- des questions courtes
- des questions de raisonnement
- des scénarios proches des cas Big Data et Cybersécurité
- des flashcards pour les notions clés

Important :
Les questions de l'application ne sont **pas** les questions exactes de l'examen officiel.

## Mode 1 — Simulation examen

À utiliser quand vous voulez vous placer en conditions proches de l'épreuve.

### Ce que vous obtenez

- un timer de `2h`
- une structure proche de l'examen
- aucune correction avant la fin
- la possibilité de :
  - passer une question
  - revenir en arrière
  - marquer une question `à revoir`

### Bon usage

- faites au moins une simulation complète avant l'examen
- gardez 10 à 15 minutes à la fin pour relire
- utilisez ensuite `Mes erreurs` pour retravailler les points faibles

## Mode 2 — Répondre et apprendre

À utiliser pour progresser rapidement sur un thème précis.

### Ce que l'application affiche

- la question
- vos choix
- la bonne réponse en vert
- les choix risqués ou faux en rouge
- une explication option par option
- la notion à retenir

### Quand l'utiliser

- quand vous découvrez encore IAM, S3 ou CloudTrail
- quand vous confondez deux notions proches
- quand vous voulez refaire vos erreurs sans refaire toute une simulation

## Mode 3 — Révision rapide

À utiliser pour des sessions courtes.

Chaque carte contient :
- un terme
- une définition
- un exemple AWS
- un piège fréquent

Exemples de notions :
- `IAM Role`
- `MFA`
- `Explicit Deny`
- `CloudTrail`
- `CloudWatch`
- `IAM Identity Center`
- `PII`

## Mode 4 — Historique

L'historique vous permet de revoir :
- vos anciennes sessions
- votre note /20
- votre score brut
- votre durée
- vos erreurs dominantes

## Mode 5 — Mes erreurs

Cette page rassemble les questions sur lesquelles vous avez été partiellement correct ou incorrect.

Utilisez-la pour :
- réviser plus efficacement
- éviter de refaire toujours les mêmes questions faciles
- concentrer vos 4 jours de préparation sur ce qui compte vraiment

## Comment lire vos résultats

Ne regardez pas uniquement la note finale.

Regardez aussi :
- le score par section
- le score par catégorie
- les notions maîtrisées
- les notions à revoir

Un résultat moyen peut cacher :
- un bon niveau en IAM
- mais une faiblesse sur les scénarios S3, Glue ou audit

## Plan de préparation conseillé sur 4 jours

### Jour 1

- relire le support principal
- faire des flashcards
- commencer par des questions courtes IAM / S3 / CloudTrail

### Jour 2

- travailler `Répondre et apprendre`
- cibler vos premières erreurs
- revoir les différences :
  - IAM User vs IAM Role
  - CloudTrail vs CloudWatch
  - RBAC vs ABAC

### Jour 3

- faire une simulation complète
- analyser les catégories faibles
- refaire uniquement vos erreurs

### Jour 4

- révision légère
- flashcards
- une courte session ciblée
- pas de surcharge de dernière minute

## Rappel méthodologique

Le bon objectif n'est pas de mémoriser des lettres.

Le bon objectif est de savoir raisonner sur :
- l'identité concernée
- la ressource visée
- l'action demandée
- le principe du moindre privilège
- les journaux utiles
- le risque de surexposition
